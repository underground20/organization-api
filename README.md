# Organization API

Сервис для управления иерархической структурой подразделений и сотрудников организации.

## Основные возможности

- **Создание, получение, изменение и удаление подразделений (Departments)**.
- **Поддержка древовидной структуры**: подразделения могут быть вложены друг в друга.
- **Создание сотрудников (Employees)** с привязкой к определенному подразделению.
- **Безопасное удаление**:
  - `cascade`: удаление подразделения со всеми дочерними элементами и сотрудниками.
  - `reassign`: удаление подразделения с переводом дочерних элементов и сотрудников в другое подразделение.
- **Валидация данных** на уровне API.

## Технологии

- **Go**: язык программирования.
- **Gin**: HTTP-фреймворк.
- **PostgreSQL**: реляционная база данных.
- **GORM**: ORM для работы с базой данных.
- **Goose**: инструмент для управления миграциями базы данных.
- **Docker & Docker Compose**: для контейнеризации и развертывания приложения.

## Запуск проекта

Для развертывания проекта используется Docker Compose, который поднимает сам сервис, базу данных PostgreSQL и мигратор.

### Требования

- Docker
- Docker Compose

### Инструкция по запуску

1. **Соберите и запустите контейнеры**:
   Выполните команду в корневой директории проекта:
   ```sh
   docker-compose up -d --build
   ```

2. **Проверка работы**:
   После успешного запуска API будет доступно по адресу `http://localhost:8080` (или по порту, который вы указали в `HOST_PORT` в вашем `.env` файле).

### Остановка проекта

Для остановки всех сервисов нажмите `Ctrl+C` в терминале, где запущен `docker-compose`, или выполните команду в другой вкладке терминала:
```sh
docker-compose down
```
